# 命询

命询是一款基于八字命盘的 AI 问答应用。用户填写出生年月日、时辰与性别后，系统会自动排出四柱、大运、流年、喜用神等信息，再结合 DeepSeek 模型生成更贴近个人命盘的问答回复与建议。

当前版本的体验重点不是“传统工具页”，而是更接近一款完整的命理咨询产品：

- 顶部导航分为 `四柱`、`问答`、`建议`
- 四柱页展示命盘矩阵、当前大运、流年与大运排布
- 问答页支持直接在前端填写并保存 `DeepSeek API Key`
- 建议页提供 `运势总述`、`工作及城市建议`、`佩戴建议（五行水晶）`

## 主要功能

### 1. 四柱命盘展示

- 根据出生信息自动计算年柱、月柱、日柱、时柱
- 展示当前大运、当前流年
- 高亮当前所处大运
- 展示喜用神、忌神与大运排布

### 2. AI 命理问答

- 用户可以像聊天一样连续提问
- 每次回复都会结合当前命盘信息
- 回答风格偏“有耐心的老师”，尽量清楚、简洁、自然

### 3. 建议模块

- 运势总述
- 工作及城市建议
- 佩戴建议（五行水晶）

### 4. 前端配置 API Key

- 问答页右上角提供 `API Key` 设置入口
- 用户可直接在前端弹窗填写 `DeepSeek API Key`
- Key 仅保存在当前浏览器本地，不会写入仓库

## 使用方式

### 1. 安装依赖

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 2. 启动服务

```bash
uvicorn app.main:app --host 127.0.0.1 --port 8000
```

如果你想局域网访问，也可以改成：

```bash
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

### 3. 打开页面

- 首页地址：`http://127.0.0.1:8000`
- API 文档：`http://127.0.0.1:8000/docs`

### 4. 首次使用流程

1. 点击右上角头像按钮，填写生辰信息
2. 切到问答页，点击右上角齿轮按钮填写 `DeepSeek API Key`
3. 开始提问，例如：

```text
我今年适合换工作吗？
接下来三个月的事业起伏怎么样？
我更适合留在现在的城市，还是考虑南方发展？
```

## API Key 说明

当前项目支持两种方式提供 `DeepSeek API Key`：

### 方式一：前端填写

推荐直接在问答页右上角齿轮按钮中填写。优点是操作简单，不需要改本地环境变量。

### 方式二：服务端环境变量

如果你更习惯后端配置，也可以使用环境变量或 `.env` 文件：

```bash
export DEEPSEEK_API_KEY="你的API Key"
```

或者在项目根目录创建 `.env`：

```env
DEEPSEEK_API_KEY=你的API Key
```

说明：

- 若前端填写了 `API Key`，聊天接口会优先使用前端传入的值
- 若前端未填写，则回退到服务端环境变量

## 项目结构

```text
ai_suanming/
├── app/
│   ├── __init__.py
│   ├── main.py
│   ├── bazi.py
│   ├── interpret.py
│   └── llm.py
├── static/
│   └── index.html
├── requirements.txt
└── README.md
```

各文件职责：

- `app/main.py`：FastAPI 入口，提供 `/api/paipan` 与 `/api/chat`
- `app/bazi.py`：八字排盘、大运流年、喜用神计算
- `app/interpret.py`：生成运势、佩戴、工作与城市建议
- `app/llm.py`：构造命盘上下文并调用 DeepSeek
- `static/index.html`：前端页面与交互逻辑

## 技术实现

### 排盘

使用 [lunar_python](https://github.com/6tail/lunar-python) 完成：

- 公历转农历
- 四柱获取
- 大运与流年计算

### 命理建议

后端会先根据命盘生成：

- 命理总述
- 工作及城市建议
- 佩戴建议（五行水晶）

这些结果会和四柱信息一起作为上下文传给模型。

### 问答逻辑

聊天接口会：

1. 解析出生信息
2. 生成命盘
3. 整理命理上下文
4. 拼接最近几轮对话历史
5. 请求 DeepSeek 返回个性化回答
